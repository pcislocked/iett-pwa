# Task List

- [ ] **Home.tsx - Concurrency & Race Conditions**: The `requestGps` callback has no debouncing or locking mechanism. Rapidly clicking the GPS retry button or overlapping initialization can spawn multiple concurrent geolocation requests and watchdog timers, potentially leading to overlapping un-debounced state updates and corrupted GPS phases.
- [ ] **AracBusOverlayPage.tsx - Silent Failure**: The `submitManualCaptcha` function catches exceptions from `fetchCaptcha()` and ignores them with a comment `// Best effort refresh only`. If fetching the next captcha fails (e.g., due to a network error), the user is left silently on the "manual-required" state with an invalid or broken captcha image.
- [ ] **client.ts - Silent Timeout Failure**: The `request<T>` method uses `AbortSignal.timeout(REQUEST_TIMEOUT_MS)`. When this triggers, `fetch` throws a `DOMException` (`AbortError`). The `isNetworkError` check only looks for `TypeError`. Consequently, `AbortError` escapes the global error handler mapping (`NETWORK_ERROR_TEXT`) and bubbles up unhandled, yielding unstyled timeout errors on the frontend.
- [ ] **Home.tsx - Swallowed Exceptions**: The `handleConsentConfirm` and `handleConsentDismiss` functions swallow `localStorage.setItem` errors without logging them.
- [ ] **usePolling.ts - State Consistency**: The `usePolling` hook reads `fetcher` from `useRef` inside the `doFetch` closure and `useEffect`. Changing the `fetcher` reference (e.g., changing the active route in `StopPage.tsx` for announcements or swapping `dcode` for stop routes) does not trigger an immediate re-fetch. The new fetcher is only picked up on the next interval tick (which can be up to 1 hour away), serving entirely stale data until then.
- [ ] **StopPage.tsx - Accessibility & Focus Trapping**: The `BusDetailSheet` component operates as a modal overlay but does not trap keyboard focus (`Tab` / `Shift+Tab`). Screen reader and keyboard users can inadvertently navigate out of the modal into the blurred background content.
- [ ] **StopPage.tsx - Accessibility & Focus Trapping**: The `InfoModal` component operates as a modal overlay but does not trap keyboard focus (`Tab` / `Shift+Tab`).
- [ ] **Test Hermeticity - API Tests**: Test suites like `client.test.ts` manually stub `global.fetch` using `vi.stubGlobal('fetch')`. There is no global hermeticity guard (e.g., `MSW` throwing on unhandled requests). If a developer forgets to mock a new endpoint, the test suite could silently trigger real network requests to the production API.

- [ ] **Home.tsx - GPS Watchdog Deadlock**: The `watchdogId` resets the UI phase to `'unavailable'` after 20s if GPS hangs, but it fails to reset `isRequestingGpsRef.current = false`. Because the hanging `getCurrentPosition` promise never settles, the `finally` block is bypassed, permanently locking the user out of retrying GPS requests.
- [ ] **AracBusOverlayPage.tsx - Captcha Refresh Silent Failure**: The inner catch block in `submitManualCaptcha` still swallows `fetchCaptcha` errors with an empty catch block (`// Best effort refresh only.`). If the subsequent captcha refresh fails due to a network error, the user is left silently on the 'manual-required' state with an invalid or stale captcha image.
- [ ] **client.ts - Futile Timeout Retry**: The `request<T>` method attempts to retry `execute(path)` if an `AbortError` (timeout) occurs. However, it reuses the exact same `requestInit` object containing the already-aborted `AbortSignal`. Fetching with an aborted signal causes `fetch` to instantly throw `DOMException: The user aborted a request`, making the retry futile.
- [ ] **StopPage.tsx - Flawed Focus Trap**: The newly added `handleKey` focus trap in `BusDetailSheet` only redirects focus if `document.activeElement` is exactly the first or last focusable element. If focus escapes to the `body` or a non-focusable node inside the modal, pressing `Tab` tabs through the background content, completely defeating the trap.
- [x] **MapPage.tsx - Accessibility & Keyboard Trap**: The "Route filter" and "Kapino / plate chip filter" dropdowns use a global `mousedown` listener instead of `onBlur`. This breaks keyboard accessibility because tabbing away from the inputs does not trigger `mousedown`, leaving the dropdowns permanently open for keyboard users.
- [x] **MapPage.tsx - Accessibility & Focus Trapping**: The `showErrorModal` escape listener uses a global `keydown` event but completely fails to implement a focus trap. Keyboard users can easily tab out of the modal into the underlying map and background elements, despite the `aria-modal="true"` attribute.
- [x] **MapPage.tsx - Concurrency & Race Condition**: The `fetchBuses` function shares a mutable `inFlight.current` Set across `useEffect` executions. When `selectedRoutes` changes, the effect is torn down (setting `alive = false` for the old closure), but the requests remain in the `inFlight` Set. The new effect skips fetching these routes. When the old requests resolve, they check the stale `alive=false` flag and abort without state updates, silently failing until the next 15-second tick.
- [x] **client.ts - Swallowed Exceptions**: In `request<T>`, the catch block intercepts network errors and indiscriminately throws `new Error(NETWORK_ERROR_TEXT)`. This swallows the original `TypeError` or `AbortError`, destroying the stack trace and masking systemic failures (like CORS or malformed URLs) from developers.
- [x] **MapPage.tsx - Time & State Consistency**: The `last_seen` timestamp on the `BusPosition` object is parsed naively using `new Date(b.last_seen)`, which parses timezone-less strings as local time. Meanwhile, `fleetMeta.updated_at` correctly uses the `parseIsoDate` helper (which explicitly enforces UTC via `Z`). This mismatch causes sorting and cache-retention regressions.
