# Task List

- [x] **Home.tsx - Concurrency & Race Conditions**: The `requestGps` callback has no debouncing or locking mechanism. Rapidly clicking the GPS retry button or overlapping initialization can spawn multiple concurrent geolocation requests and watchdog timers, potentially leading to overlapping un-debounced state updates and corrupted GPS phases.
- [x] **AracBusOverlayPage.tsx - Silent Failure**: The `submitManualCaptcha` function catches exceptions from `fetchCaptcha()` and ignores them with a comment `// Best effort refresh only`. If fetching the next captcha fails (e.g., due to a network error), the user is left silently on the "manual-required" state with an invalid or broken captcha image.
- [x] **client.ts - Silent Timeout Failure**: The `request<T>` method uses `AbortSignal.timeout(REQUEST_TIMEOUT_MS)`. When this triggers, `fetch` throws a `DOMException` (`AbortError`). The `isNetworkError` check only looks for `TypeError`. Consequently, `AbortError` escapes the global error handler mapping (`NETWORK_ERROR_TEXT`) and bubbles up unhandled, yielding unstyled timeout errors on the frontend.
- [x] **Home.tsx - Swallowed Exceptions**: The `handleConsentConfirm` and `handleConsentDismiss` functions swallow `localStorage.setItem` errors without logging them.
- [x] **usePolling.ts - State Consistency**: The `usePolling` hook reads `fetcher` from `useRef` inside the `doFetch` closure and `useEffect`. Changing the `fetcher` reference (e.g., changing the active route in `StopPage.tsx` for announcements or swapping `dcode` for stop routes) does not trigger an immediate re-fetch. The new fetcher is only picked up on the next interval tick (which can be up to 1 hour away), serving entirely stale data until then.
- [x] **StopPage.tsx - Accessibility & Focus Trapping**: The `BusDetailSheet` component operates as a modal overlay but does not trap keyboard focus (`Tab` / `Shift+Tab`). Screen reader and keyboard users can inadvertently navigate out of the modal into the blurred background content.
- [x] **StopPage.tsx - Accessibility & Focus Trapping**: The `InfoModal` component operates as a modal overlay but does not trap keyboard focus (`Tab` / `Shift+Tab`).
- [x] **Test Hermeticity - API Tests**: Test suites like `client.test.ts` manually stub `global.fetch` using `vi.stubGlobal('fetch')`. There is no global hermeticity guard (e.g., `MSW` throwing on unhandled requests). If a developer forgets to mock a new endpoint, the test suite could silently trigger real network requests to the production API.

- [x] **Home.tsx - GPS Watchdog Deadlock**: The `watchdogId` resets the UI phase to `'unavailable'` after 20s if GPS hangs, but it fails to reset `isRequestingGpsRef.current = false`. Because the hanging `getCurrentPosition` promise never settles, the `finally` block is bypassed, permanently locking the user out of retrying GPS requests.
- [x] **AracBusOverlayPage.tsx - Captcha Refresh Silent Failure**: The inner catch block in `submitManualCaptcha` still swallows `fetchCaptcha` errors with an empty catch block (`// Best effort refresh only.`). If the subsequent captcha refresh fails due to a network error, the user is left silently on the 'manual-required' state with an invalid or stale captcha image.
- [x] **client.ts - Futile Timeout Retry**: The `request<T>` method attempts to retry `execute(path)` if an `AbortError` (timeout) occurs. However, it reuses the exact same `requestInit` object containing the already-aborted `AbortSignal`. Fetching with an aborted signal causes `fetch` to instantly throw `DOMException: The user aborted a request`, making the retry futile.
- [x] **StopPage.tsx - Flawed Focus Trap**: The newly added `handleKey` focus trap in `BusDetailSheet` only redirects focus if `document.activeElement` is exactly the first or last focusable element. If focus escapes to the `body` or a non-focusable node inside the modal, pressing `Tab` tabs through the background content, completely defeating the trap.
- [x] **MapPage.tsx - Accessibility & Keyboard Trap**: The "Route filter" and "Kapino / plate chip filter" dropdowns use a global `mousedown` listener instead of `onBlur`. This breaks keyboard accessibility because tabbing away from the inputs does not trigger `mousedown`, leaving the dropdowns permanently open for keyboard users.
- [x] **MapPage.tsx - Accessibility & Focus Trapping**: The `showErrorModal` escape listener uses a global `keydown` event but completely fails to implement a focus trap. Keyboard users can easily tab out of the modal into the underlying map and background elements, despite the `aria-modal="true"` attribute.
- [x] **MapPage.tsx - Concurrency & Race Condition**: The `fetchBuses` function shares a mutable `inFlight.current` Set across `useEffect` executions. When `selectedRoutes` changes, the effect is torn down (setting `alive = false` for the old closure), but the requests remain in the `inFlight` Set. The new effect skips fetching these routes. When the old requests resolve, they check the stale `alive=false` flag and abort without state updates, silently failing until the next 15-second tick.
- [x] **client.ts - Swallowed Exceptions**: In `request<T>`, the catch block intercepts network errors and indiscriminately throws `new Error(NETWORK_ERROR_TEXT)`. This swallows the original `TypeError` or `AbortError`, destroying the stack trace and masking systemic failures (like CORS or malformed URLs) from developers.

## Frontend Review:
- [x] **MapPage.tsx - Performance / Memory Leak (CanvasMarkers)**:  usMarkers array is defined dynamically on every render, causing <CanvasMarkers markers={busMarkers} /> to trigger its useEffect continuously. This forces Leaflet to clear and recreate all canvas markers on every keystroke or map state change, completely defeating the purpose of canvas rendering and causing memory churn. The array must be memoized via useMemo.
- [x] **Hooks / MapPage.tsx - Type Downgrade**: In useFleet.ts, the mapQuery utility lacks generics, returning data: any. This stripped all type safety from the fleet list, causing the PR author to defensively cast (b: any) inside all array map/filter functions in MapPage.tsx. mapQuery should use generics (e.g., function mapQuery<T, E>(query: UseQueryResult<T, E>)).
- [x] **useArrivals.ts - Missing enabled check**: The refactored useArrivals hook executes the useQuery even when dcode is empty, whereas useRouteBuses correctly uses enabled: !!hatKodu. It needs enabled: !!dcode to prevent invalid network requests when the stop code is missing.
- [x] **CanvasMarkers.tsx - Event Listener Leak**: The useEffect cleanup iteratively calls layer.removeMarker(m), but it never unbinds the marker.off('click') event listeners that were manually attached, which can lead to memory leaks over time as markers are replaced.
- [x] **useFleet.ts - Stale Data Regression**: `useFleet` incorrectly maps `query.isStale` to the hook's `stale` flag. With a global `staleTime` of 10 seconds, this causes the data to be marked as stale 10 seconds after every fetch. The original intent (and the behavior of the other hooks) was to mark data as stale ONLY when a fetch fails but previous data is retained (`query.isError && !!query.data`). This bug causes `MapPage.tsx` to show false stale warnings.
- [x] **MapPage.tsx - Incompatible Marker Icons (CanvasMarkers)**: `leaflet-canvas-marker` only supports `L.Icon` with a valid `iconUrl`. `MapPage.tsx` passes `busIcon`, which is an `L.divIcon` relying on HTML markup. The canvas renderer cannot draw HTML elements directly, resulting in buses failing to render visually on the canvas.- [x] **StopPage.tsx - State Consistency & UX Regression**: The `AutoFitBuses` component's `useEffect` includes `buses.length` in its dependency array. Every time background polling updates the number of buses, the map aggressively triggers `map.flyToBounds` or `map.flyTo`, forcefully interrupting and overwriting the user's manual pan/zoom interactions. The auto-fit should only fire on initial load or when `filterKey` changes.
- [x] **StopPage.tsx - Concurrency & Rendering Performance**: The drag-to-resize handle for the map calls `setMapHeightPct` inside `onHandlePointerMove`. Since this state resides at the top level of `StopPage`, it triggers a full React re-render of the entire component tree (including lists, maps, and hooks) at the browser's pointer event rate, causing severe UI jank. A mutable ref should be used to update the DOM element's style directly during drag.
- [x] **StopPage.tsx - State Consistency & Animation Jank**: The `filteredArrivals.map` for the `AnimatePresence` `motion.div`s uses a fallback `key` combining properties and the array index `i` (`key={\`...-${a.kapino || i}\`}`). Since the list is dynamically sorted by ETA, an item dropping out shifts the index `i` for all subsequent items lacking a `kapino`. This breaks React's reconciliation, causing Framer Motion's `layout` transitions to fail and items to visually flash.
- [x] **StopPage.tsx - Error Handling & Silent Failures**: Inside `annsFetcher`, if a subset of the announcement API requests fail but others succeed, the `failedRequests > 0` block silently swallows the failures unless *all* requests fail. The user is left completely unaware that announcements for certain routes failed to load.
- [x] **StopPage.tsx - Accessibility & UX Architecture**: The `BusDetailSheet` implements a keyboard focus trap via a 'keydown' listener, but fails to set `aria-hidden="true"` on the underlying page content. Screen reader users navigating with virtual cursors can easily escape the dialog and read the main page content beneath the backdrop, causing contextual confusion.
- [x] **StopPage.tsx - Resource Management (DOM Churn)**: `routeIconMap` creates new `L.divIcon` objects inside a `useMemo` that depends on `arrivalRouteOrder`. Whenever the ETA-based arrival order changes, entirely new Leaflet icon instances are generated. Passing new icon references to `<Marker>` forces Leaflet to unnecessarily destroy and recreate the marker's internal DOM node, adding GC pressure and potential micro-stutters.
- [x] **StopPage.tsx - Resource Management Leak**: `annsFetcher` ignores the `AbortSignal` provided by React Query's `queryFn`. If the user navigates away from the page while the sequential `for` loop is processing chunks of network requests, the loop continues executing in the background, wasting bandwidth and connections.

- [x] **client.ts - API Abortion Leak**: The entire `api` object (e.g., `api.routes.stops(hatKodu: string)`) drops the `init?: RequestInit` parameter before passing to `get()`. This makes it impossible for React Query to pass an `AbortSignal` to the underlying `fetch` call. If a user navigates away, inflight HTTP requests are orphaned in the background, leaking bandwidth and exhausting browser connection limits.
- [x] **NearbyPage.tsx - Unbounded Network Concurrency**: `fetchNearby` uses `Promise.allSettled(nearby.map((s) => api.stops.routes(s.stop_code)))` to enrich stop data. This fires simultaneous network requests for every single stop returned by the API (which can be dozens), severely degrading client performance and causing concurrent request amplification (DDOS risk) against the backend. Enrichment should be chunked or limited to the nearest `N` stops.
- [x] **NearbyPage.tsx - Resource Management Leak (RAF)**: The `scrollRafRef` variable tracks pending `requestAnimationFrame` calls to debounce scroll events, but there is no `useEffect` cleanup routine to call `cancelAnimationFrame(scrollRafRef.current)` on unmount. A pending frame can execute after the component unmounts, firing stale state updates and causing memory leaks.
- [x] **useFavorites.ts - Swallowed/Unhandled Exceptions**: The `save` function synchronously calls `localStorage.setItem(KEY, JSON.stringify(favs))` without a `try/catch` guard. If `localStorage` is quota-exceeded or blocked in strict privacy modes, the exception escapes the `toggle` callback, instantly crashing the React render cycle instead of degrading to transient in-memory state.
